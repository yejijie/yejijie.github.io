Class文件装载流程

装载Class类型可以分为加载、连接、和初始化三个步骤。其中连接又可分为验证、准备和解析3步。

加载

加载类时，JVM完成以下工作：

  • 通过类的全民，获取类的二进制数据流
  • 解析类的二进制数据流为方法区内的数据结构
  • 创建java.lang.Class类的实例，表示该类型

连接

验证

目的是保证加载的类的字节码是合法、合理并符合规范的。包括：

  • 格式检查: 魔数检查，版本检查、长度检查
  • 语义检查：是否继承final、是否有父类、抽象方法是否有实现
  • 字节码验证：跳转指令是否指向正确位置、操作数类型是否合理
  • 符号引用验证：符号引用的直接引用是否存在

准备

分配内存空间，并设置初始值。
jvm为各类型变量默认的初始值如下表：

类型 默认初始值
int 0
long 0L
short (short)0
char \u0000
boolean false
reference null
float 0f
double 0f

解析

将符号引用转为直接引用，也就是得到类或者字段、方法在内存中的指针或者偏移量。

初始化

初始化阶段的重要工作是执行类的初始化方法。方法是由编译器自动生成的，它是由类静态成员的赋值语句以及static语句块合并产生的。
